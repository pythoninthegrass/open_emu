Create a new issue following the project's hygiene rules.

1. Search for duplicates first:
   gh issue list --repo nickybmon/OpenEmu-Silicon --state open
   Ask the user to confirm no duplicate exists before proceeding.

2. Ask: what is the issue type? (bug / feature / core integration / checklist)

3. Draft a title. Rules:
   - No type prefixes (no fix:, feat:, note:, bug: in the title)
   - Describes the problem, not the solution
   - Format: `[Component] — [what is wrong or missing]`

4. Select the correct template based on type.

5. Apply appropriate labels.

6. Create the issue and report the issue number and URL.
